# i18n scaffolding

**Idea:** publish `lang/`, share translations to React through
`HandleInertiaRequests` (or adopt `laravel-react-i18n`), demo the convention on
one auth page. Owner builds for Turkish/Spanish markets — retrofitting i18n
after ten hardcoded pages is exactly the rework the token policy exists to avoid.

**Open question:** which convention (shared props vs. dedicated package) — decide
on the first non-English project, then graduate.
