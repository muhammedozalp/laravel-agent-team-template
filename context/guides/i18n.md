# i18n — Turkish default, extensible (en/ar/…)

_Zero-dependency by design (2026-07 research): the once-standard
`laravel-react-i18n` package is unmaintained and broken on React 19, so
translations flow from Laravel's `lang/` through a shared Inertia prop into a
~70-line TS helper. Single source of truth; backend validation messages use
the same files automatically._

## How it works

| Piece | Where |
|---|---|
| Default locale **`tr`**, fallback `en`; allowed list | `config/app.php` (`locale`, `available_locales`) |
| Locale resolution: user's saved choice → session → default | `app/Http/Middleware/SetLocale.php` (web group) |
| Switching: `POST /locale` (guests → session; users → `users.locale`) | `LocaleController`, route `locale.update` |
| Translations shared to React (cached per locale) | `HandleInertiaRequests::share()` — `locale` + `translations` props |
| Frontend helpers `t()` / `tChoice()` | `resources/js/lib/i18n.ts` — `const { t } = useTranslations()` |
| Turkish strings (250+ common + validation) | `lang/tr.json`, `lang/tr/*.php` (laravel-lang) |
| RTL readiness | `dir` attribute in `app.blade.php` flips for `ar/fa/he/ur` |

## Working rules

- **Never hardcode user-facing strings in TSX** — `t('Log in')` from day one
  (see `welcome.tsx` nav). English source text as the key, Laravel-style.
- Backend strings: `__('...')` as usual — same `lang/` files.
- Plurals: Laravel `trans_choice` syntax works in `tChoice()` (`{0} none|[1,*] :count items`);
  plain `one|many` pairs pluralize via `Intl.PluralRules`, so Arabic's six forms
  resolve correctly.
- Params: `:name` / `:NAME` interpolation, identical to Laravel.

## Adding a language (e.g. Arabic)

1. `docker compose exec app php artisan lang:add ar` (laravel-lang strings).
2. Add `'ar'` to `available_locales` in `config/app.php`.
3. Translate your project's own keys in `lang/ar.json`.
4. RTL is automatic (`dir="rtl"`); use Tailwind **logical** utilities
   (`ms-*/me-*/ps-*/pe-*/text-start`) instead of `ml-/mr-` in new components.
5. Public SEO in a second language → that's the point to add `/en`, `/ar` URL
   prefixes + `hreflang` (deliberately not built until needed — session-based
   switching has no SEO surface).

## Cache note

Shared translations cache forever per locale (`translations.{locale}`) — after
editing `lang/` in dev run `php artisan cache:clear` (deploys clear it via the
release script's cache rebuild).
